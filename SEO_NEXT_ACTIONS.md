# SEO改善後の次のアクション

## ✅ 完了した作業

1. **メタデータの拡充**
   - ルートレイアウトと各ページのキーワードを大幅に拡充
   - ターゲット1-4向けのキーワードを網羅的に追加
   - 英語、カタカナ、ひらがな、漢字の多様な表記に対応

2. **構造化データの追加**
   - FAQPageスキーマ（6つのよくある質問）
   - BreadcrumbList（各ページ）
   - CollectionPageスキーマ（各一覧ページ）
   - NewsArticleスキーマ（ニュースページ）

3. **hreflangタグの追加**
   - 日本語・英語対応

4. **Open GraphとTwitterカードの最適化**
   - 各ページに適切な画像と説明を追加

## 🚀 次のアクション（優先順位順）

### 1. 即座に実施すべきこと

#### ✅ 変更内容のデプロイ
```bash
# 変更をコミット
git add .
git commit -m "SEO改善: メタデータ拡充、構造化データ追加、hreflang対応"

# 本番環境にデプロイ（Vercelの場合）
git push origin main
```

#### ✅ デプロイ後の動作確認
1. **メタデータの確認**
   - ブラウザの開発者ツールで各ページの`<head>`を確認
   - メタタグが正しく表示されているか確認
   - 構造化データ（JSON-LD）が正しく出力されているか確認

2. **構造化データの検証**
   - [Google リッチリザルトテスト](https://search.google.com/test/rich-results)で各ページをテスト
   - エラーがないことを確認

3. **サイトマップの確認**
   - `https://db.nexanahq.com/sitemap.xml` にアクセス
   - 正しく生成されているか確認

### 2. Google Search Consoleでの作業

#### ✅ サイトマップの送信（未実施の場合）
1. Google Search Consoleにアクセス
2. 「サイトマップ」セクションに移動
3. `https://db.nexanahq.com/sitemap.xml` を追加
4. 「送信」をクリック

#### ✅ 主要ページのインデックス登録リクエスト
以下の主要ページでインデックス登録をリクエスト：
- `https://db.nexanahq.com/`
- `https://db.nexanahq.com/contests`
- `https://db.nexanahq.com/open-calls`
- `https://db.nexanahq.com/workspace`
- `https://db.nexanahq.com/news`

**手順：**
1. Google Search Consoleの「URL検査」ツールを使用
2. 各URLを入力
3. 「インデックス登録をリクエスト」をクリック

#### ✅ URL検査の実行
1. 「URL検査」ツールで各主要ページを検査
2. 「ライブテスト」を実行
3. エラーがないことを確認
4. 構造化データが正しく認識されているか確認

### 3. 1週間以内に実施すべきこと

#### 📊 インデックス状況の監視
1. **Google Search Consoleで確認**
   - 「インデックス作成 > ページ」セクションでインデックス状況を確認
   - 「有効」ページ数が増えているか確認

2. **検索結果での確認**
   - Googleで `site:db.nexanahq.com` と検索
   - インデックスされたページが表示されるか確認

3. **パフォーマンスの確認**
   - 「パフォーマンス」セクションで検索結果での表示回数・クリック数を確認
   - 主要キーワードでの表示状況を確認

#### 🔍 構造化データの監視
1. Google Search Consoleの「拡張」セクションで確認
2. 構造化データのエラーがないか確認
3. エラーがある場合は修正

### 4. 継続的に実施すべきこと

#### 📝 コンテンツの定期更新
- 新しいコンテスト、公募、ニュースを定期的に追加
- 更新頻度が高いほど、Googleのクロール頻度も上がります

#### 🔗 内部リンクの最適化
- トップページから各セクションへのリンクが適切に配置されているか確認
- 関連ページ間の内部リンクを充実させる

#### 📈 SEOパフォーマンスの監視
- 月1回、Google Search Consoleで以下を確認：
  - インデックス登録されたページ数
  - 検索結果での表示回数・クリック数
  - 主要キーワードでのランキング
  - エラーや警告の有無

## 🛠️ トラブルシューティング

### 構造化データのエラーが出る場合
1. [Google リッチリザルトテスト](https://search.google.com/test/rich-results)でエラーを確認
2. エラーの内容に応じてコードを修正
3. 再デプロイ後、再度テスト

### インデックス登録されない場合
1. robots.txtでブロックされていないか確認
2. サイトマップが正しく送信されているか確認
3. ページの品質（コンテンツの充実度）を確認
4. 数日待ってから再度確認（初回は時間がかかることがあります）

### メタデータが表示されない場合
1. ブラウザのキャッシュをクリア
2. デプロイが完了しているか確認
3. 開発者ツールでHTMLソースを確認
